Overview

The Challenge

The client needed a modern e-commerce platform that could serve customers across Saudi Arabia while maintaining full compliance with ZATCA e-invoicing regulations. The solution required seamless integration with multiple payment providers and real-time delivery tracking capabilities.

The existing system was outdated, lacked mobile presence, and wasn't compliant with new Saudi tax regulations. Users were abandoning carts due to complicated checkout processes and limited payment options.

  • Mobile-first approach for the Saudi market
  • ZATCA Phase 2 e-invoicing compliance
  • Buy Now, Pay Later integration with Tabby
  • Real-time tracking with Google Maps

Solution

The Approach

I architected a scalable full-stack solution using Flutter for cross-platform mobile development and Laravel for a robust backend API. The modular architecture ensures easy maintenance and future feature additions.

Special attention was given to the checkout flow optimization, reducing steps from 7 to 3, and implementing one-tap payments for returning customers.

  • Clean Architecture with separation of concerns
  • State Management using Provider/Riverpod
  • RESTful APIs with Laravel Sanctum auth
  • Caching strategies for optimal performance

Problem Solving

Challenges & Solutions

ZATCA Compliance
Challenge: Implementing complex ZATCA Phase 2 e-invoicing requirements with QR codes and digital signatures.

Solution: Built a dedicated ZATCA service module with automatic XML generation, QR code embedding, and real-time validation against ZATCA servers.
Multiple Payment Gateways
Challenge: Integrating three different payment providers with varying APIs and response formats.

Solution: Created a unified payment abstraction layer allowing seamless switching between gateways while maintaining consistent user experience.
Real-time Sync
Challenge: Keeping inventory and order status synchronized across mobile app and admin panel.

Solution: Implemented WebSocket connections with Firebase for real-time updates and offline-first architecture with local caching.
Performance Optimization
Challenge: App was initially slow with large product catalogs and images.

Solution: Implemented lazy loading, image compression, pagination, and Redis caching on the API side achieving 40% faster load times.
